# Mission: Read and debug Flutter from a JavaScript background

## Why
Use existing JavaScript experience to trace real Flutter application behavior, diagnose API flows, and confidently maintain the JAMORE mobile app.

## Success looks like
- Trace a user action from the widget callback through state, repository, and HTTP layers
- Predict where `async` functions pause and what each `Future<T>` resolves to
- Place breakpoints and inspect success and error values without relying on broad logging

## Constraints
- Teach through the existing JAMORE codebase
- Explain in Thai and map unfamiliar Dart concepts to JavaScript concepts
- Keep each lesson short and immediately practical

## Out of scope
- Advanced Flutter rendering and performance internals
- Native Android and iOS implementation details
