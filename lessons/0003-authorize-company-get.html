Authorize Company/Get ด้วย TokenUniverse

เป้าหมาย: เรียก protected endpoint โดยแยก “ผู้ใช้คือใคร” ออกจาก “กำลังอ่าน tenant ไหน”

1. อ่าน backend contract จาก attribute

[Authorize] อยู่บน CompanyController จึงครอบทุก action

GetUserCompany มี [AllowAnonymous] จึงไม่บังคับ token

Get/{CompanyID} ไม่มี [AllowAnonymous] จึงต้องผ่าน JWT Bearer validation

2. TokenUniverse ถูกต่อเข้าระบบอยู่แล้ว

AuthRepository.login อ่าน tokenUniverse มาเก็บใน AuthSession.token จากนั้น ApiClient._headers อ่าน token ผ่าน callback และเติม Authorization: Bearer ... ทุก request

// Feature repository ไม่ต้องรู้จัก token
return _apiClient.get('Company/Get/$encodedCompanyId');

3. แต่ endpoint นี้ต้องมี tenant header ด้วย

Angular เดิมส่งทั้ง Authorization และ x-companyid ส่วน backend ใช้ x-companyid ใน EF Core global query filter ดังนั้น Flutter ต้องรักษาพฤติกรรมนี้

Future<Object?> getCompany(String companyId) {
  final normalized = companyId.trim();
  final encoded = Uri.encodeComponent(normalized);

  return _apiClient.get(
    'Company/Get/$encoded',
    headers: {'x-companyid': normalized},
  );
}
อย่าใส่ Authorization ที่นี่: ถ้าแต่ละ repository จัดการ token เอง โค้ดจะซ้ำและ token refresh/logout จะดูแลยาก ให้ network layer เป็นเจ้าของ authentication header

4. เรียกหลัง login

final session = await _authGateway.login(...);
final companyId = session.companyId?.toString().trim();

if (companyId == null || companyId.isEmpty) {
  throw const AuthException('Company ID is missing.');
}

final companyResponse = await _companyGateway.getCompany(companyId);
// breakpoint: inspect companyResponse

5. แบบฝึกตรวจของจริง

  1. เพิ่ม getCompany ใน interface, implementation และ fake test gateway
  2. เรียกด้วย session.companyId หลัง login
  3. วาง breakpoint ที่ ApiClient._headers
  4. ยืนยันว่ามีทั้ง Authorization และ x-companyid
  5. ทดสอบลบ Authorization ชั่วคราว ควรได้ 401 แล้วคืนโค้ดกลับ
