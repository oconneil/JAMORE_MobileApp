Trace Login แบบ JavaScript developer

เป้าหมาย: มอง Login เป็น promise chain ที่แยกความรับผิดชอบเป็น 4 ชั้น และตามค่ากลับขึ้นมาได้

1. เริ่มจากสิ่งที่คุ้นเคย

Dart Future<T> ทำหน้าที่เดียวกับ JavaScript Promise<T> และ await ให้ค่าที่สำเร็จหรือโยน error เข้าสู่ catch เช่นเดียวกัน

// JavaScript
async function login() {
  const response = await api.post('/login');
  return response;
}

// Dart
Future<Object?> login() async {
  final response = await api.post('/login');
  return response;
}

2. อ่าน signature ก่อน body

Future<bool> login(...) แปลว่า “ตอนนี้ได้ Future กลับมา และภายหลังจะจบด้วย bool หรือ error” ชนิดที่อยู่ใน <...> คือค่าหลัง await

3. Login จริงเป็น call chain

UI: รับ event จากปุ่มและแสดง loading/error

State: ตัดสินว่า login สำเร็จไหม และเปลี่ยนหน้า

Repository: แปลง JSON ให้เป็น AuthSession

API client: ส่ง HTTP, ตรวจ status, decode JSON

ใน debugger ให้เริ่มที่ callback ของปุ่ม แล้วกด Step Into ตรงแต่ละ await เมื่อ response กลับมาให้กด Step Over เพื่อดูว่าค่าถูกแปลงอย่างไร จากนั้นดู Call Stack เพื่อรู้ว่ามาจากชั้นใด

4. แบบฝึก 2 นาที

ก่อนรัน ให้ทำนายค่าหลังแต่ละจุด:

  1. ApiClient.request คืนค่าอะไรเมื่อ status 200?
  2. AuthRepository.login คืนค่าอะไรเมื่อมี token?
  3. AppState.login คืนค่าอะไร และ state ไหนเปลี่ยน?
  4. ถ้า status 401 control จะวิ่งเข้า catch จุดไหน?

จากนั้นวาง breakpoint แล้วพิสูจน์คำตอบ การทำนายก่อนดูคือส่วนที่ทำให้ mental model แม่นขึ้น
