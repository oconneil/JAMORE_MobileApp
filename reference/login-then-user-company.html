Login → GetUserCompany

เป้าหมาย: Login ให้ได้ token ก่อน แล้วค่อยเรียก endpoint ที่ต้องใช้ token แบบลำดับต่อกัน

ลำดับ control flow

await auth.login()
↓ AuthRepository เก็บ session/token
await company.getUserCompany(session.userName)
↓ ApiClient ขอ token จาก accessTokenProvider
Authorization: Bearer <TokenUniverse>
↓ ได้ company response
เก็บ state → เปลี่ยน route → notifyListeners()

Repository request

Future<Object?> getUserCompany(String userName) {
  final encoded = Uri.encodeComponent(userName.trim());
  return _apiClient.get('Company/GetUserCompany/$encoded');
}

จุดที่ต้องจำ

เรียก company หลัง await login() คืนค่าแล้ว เพราะตอนนั้น AuthRepository._session ถูกตั้งและ ApiClient จึงแนบ Bearer token ได้

อย่าเปลี่ยนหน้า dashboard ก่อน company call ถ้าข้อมูล company เป็น prerequisite ของหน้า และอย่าเดา response model—หยุด debugger ดู decoded JSON ก่อน
