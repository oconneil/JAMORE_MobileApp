# Domain Docs

This is a single-context repository.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant decisions under `docs/adr/` when they exist.
- If either location is absent, proceed without flagging it; producer skills create these files lazily.

## Vocabulary

Use domain terms exactly as defined in `CONTEXT.md`. If a required concept is absent, reconsider whether the term is needed or record the gap during a domain-design discussion.

## ADR conflicts

Surface any recommendation that contradicts an existing ADR rather than silently overriding the decision.
